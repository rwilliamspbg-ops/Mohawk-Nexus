## ⚠️ Common Pitfalls (Avoid These!)

❌ **Pitfall 1: Trying to build all 3 demos perfectly**
→ Build 2 demos well instead of 3 badly.

❌ **Pitfall 2: Forgetting to print/memorize the pitch**
→ You'll freeze. Say it out loud now.

❌ **Pitfall 3: Deploying only to localhost**
→ Push to GitHub Pages. Judges might want to explore offline.

❌ **Pitfall 4: Not testing the environment tonight**
→ You'll waste hour 1 installing Go or TinyGo.

❌ **Pitfall 5: Trying to explain Lean4 proofs**
→ Just say "Mathematically proven correct." Judges don't need implementation details.

❌ **Pitfall 6: No backup plan for live demo crashes**
→ Have screenshots ready. A video. Fallback code walkthrough.
